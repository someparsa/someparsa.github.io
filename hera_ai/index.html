Publication pathway

From research output to citable release

A shared checklist can help every team publish outputs in the same professional, reproducible format.

  1. 01

    Create a GitHub repository

    Include README, licence, contributors, installation notes, and a minimal working example.

  2. 02

    Version the output

    Tag stable releases using semantic versioning for software or date-based versions for reports.

  3. 03

    Archive with Zenodo

    Connect GitHub to Zenodo, archive each release, and record the DOI and citation text.

  4. 04

    Publish a hub record

    Add the short description, team, version, DOI, related paper, and maintenance status.
